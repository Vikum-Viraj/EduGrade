# Edugrade - Short Documentation

## 1. Overview
Edugrade is a Java Swing desktop application for managing university data with SQL Server.

Main modules:
- Department Management
- Student Management
- Lecturer Management
- Course Management
- Enrollment Management

Architecture pattern used:
- Model: entity classes in src/model
- DAO: database access in src/dao
- Controller: application flow in src/controller
- View: Swing screens in src/ui

## 2. Project Structure
- src: Java source code
- lib: external JAR files (includes SQL Server JDBC driver)
- bin: compiled class files (build output)
- .env: local runtime database configuration (not committed)

## 3. Database Configuration
Connection settings are read by src/config/DBConnection.java.

Required environment values:
- DB_USER
- DB_PASSWORD

Optional:
- DB_URL (defaults to the local SQL Server URL in the code)

You can set values in either:
- OS environment variables, or
- .env file in project root

Example .env content:
DB_URL=jdbc:sqlserver://LAPTOP-OFNPMO5R:1433;databaseName=UniDB;encrypt=false;trustServerCertificate=true
DB_USER=your_user
DB_PASSWORD=your_password

## 4. Database Schema (High-Level)
Tables used by the app:
- Departments
	- department_id (PK, identity)
	- department_name

- Students
	- student_id (PK, identity)
	- name, email, age, address, phone
	- department_id (FK to Departments)

- Lecturers
	- lecturer_id (PK, identity)
	- name, email, age, address, phone
	- department_id (FK to Departments)

- Courses
	- course_id (PK, identity)
	- course_name, credits
	- department_id (FK to Departments)

- Enrollments
	- enrollment_id (PK, identity)
	- student_id (FK to Students)
	- course_id (FK to Courses)
	- enrollment_date
	- grade

Main relationships:
- One Department to many Students, Lecturers, and Courses
- One Student to many Enrollments
- One Course to many Enrollments

## 5. Query Patterns Used
The DAO layer uses prepared statements for all operations.

Typical operations in each DAO:
- Create: INSERT with generated key
- Read single: SELECT ... WHERE id = ?
- Read all: SELECT ... ORDER BY id
- Update: UPDATE ... WHERE id = ?
- Delete: DELETE ... WHERE id = ?

Join-based display queries:
- Student list joins Departments for department_name
- Lecturer list joins Departments for department_name
- Course list joins Departments for department_name
- Enrollment list joins Students, Courses, and Departments to show combined data in the UI

## 6. Application Usage
1. Compile
	 - javac -cp .;lib/* -d .\\bin @sources.txt

2. Run
	 - java -cp .;bin;lib/* App

3. Use dashboard
	 - Open the app and choose a module
	 - Create records with the form at the top
	 - Use row buttons in tables to Update or Delete
	 - Use Refresh List to reload latest database state

## 7. Notes and Troubleshooting
- If you see DB_USER is required, create .env or set OS environment variables.
- If insert or update fails with permission denied, grant SQL permissions on UniDB for your DB user.
- IDs are auto-generated by the database; forms intentionally do not ask for IDs when creating records.
